= ProcessorApiManager - API-Based Processor Management Guide

== Overview

The `ProcessorApiManager` utility provides reliable, API-based management of processors on the NiFi canvas via the NiFi REST API, replacing UI-based interactions. It provides full lifecycle management (add, remove, start, stop) for the `MultiIssuerJWTTokenAuthenticator` and discovery/navigation support for the `RestApiGatewayProcessor`, with process group navigation and result caching.

== Key Components

=== ProcessorApiManager (`utils/processor-api-manager.js`)

Core methods (JWT processor):

* *`ensureProcessorOnCanvas()`* — Ensures JWT processor is on canvas (adds if missing). Throws on failure.
* *`verifyMultiIssuerJWTTokenAuthenticatorIsDeployed()`* — Returns `boolean`: whether the processor type is available
* *`verifyMultiIssuerJWTTokenAuthenticatorIsOnCanvas()`* — Returns `{ exists: boolean, processor: object }`: whether the processor is on the canvas
* *`addMultiIssuerJWTTokenAuthenticatorOnCanvas(position)`* — Adds the processor at specified position
* *`removeMultiIssuerJWTTokenAuthenticatorFromCanvas()`* — Removes the processor from the canvas
* *`startProcessor()`* / *`stopProcessor()`* — Start/stop the processor

Gateway and process group methods:

* *`ensureGatewayProcessorOnCanvas()`* — Verifies gateway processor exists and navigates to its process group (throws if not found)
* *`getGatewayProcessorId()`* — Finds the RestApiGatewayProcessor and returns its ID
* *`getGatewayProcessGroupId()`* — Finds the REST API Gateway process group ID
* *`getJwtPipelineProcessGroupId()`* — Finds the JWT Auth Pipeline process group ID
* *`navigateToProcessGroup(groupId)`* — Navigates the browser to a specific process group
* *`getProcessorDetails(processorId)`* — Gets detailed processor info by ID
* *`getProcessorsOnCanvas(groupId)`* — Returns all processors in a group
* *`getRootProcessGroupId()`* — Gets the root process group ID
* *`stopAllProcessors()`* — Stops all processors in the JWT Pipeline group
* *`removeConnectionsForProcessor(processorId)`* — Removes all connections to/from a processor
* *`makeApiCall(path, options)`* — Low-level NiFi REST API call (uses page context for auth)
* *`clearCache()`* (static) — Clears the module-level process group/processor ID cache

NOTE: `ProcessorApiManager` caches process group and processor IDs after first lookup (static cache persists across instances). Call `ProcessorApiManager.clearCache()` to force fresh API lookups. The cache is pre-seeded from `.auth/test-context.json` when available.

=== Setup Helper (`utils/processor-setup-helper.js`)

Simplified functions for test setup/teardown:

* *`setupMultiIssuerJWTAuthenticator(page, options)`* — Returns `true`/`false` (does not throw). Options: `{ removeFirst: boolean }`
* *`cleanupMultiIssuerJWTAuthenticator(page)`* — Removes processor after tests
* *`verifyProcessorStatus(page)`* — Gets detailed processor status

NOTE: `ProcessorApiManager` requires prior authentication — API calls use the page's browser context cookies (set during `AuthService.ensureReady()`). Without authentication, all API calls will fail with 401.

== Usage Examples

=== Basic Usage

[source,javascript]
----
import { test, expect } from '@playwright/test';
import { ProcessorApiManager } from '../utils/processor-api-manager.js';
import { AuthService } from '../utils/auth-service.js';

test('My test that needs the processor', async ({ page }) => {
  const authService = new AuthService(page);
  await authService.ensureReady();

  const processorManager = new ProcessorApiManager(page);
  await processorManager.ensureProcessorOnCanvas();

  // Your test logic here...
});
----

=== Using the Setup Helper

[source,javascript]
----
import { test } from '@playwright/test';
import { setupMultiIssuerJWTAuthenticator } from '../utils/processor-setup-helper.js';
import { AuthService } from '../utils/auth-service.js';

test.beforeEach(async ({ page }) => {
  const authService = new AuthService(page);
  await authService.ensureReady();

  const ready = await setupMultiIssuerJWTAuthenticator(page);
  if (!ready) {
    test.skip('Processor setup failed');
  }
});
----

NOTE: `setupMultiIssuerJWTAuthenticator()` returns `true`/`false` and does not throw.
`ProcessorApiManager.ensureProcessorOnCanvas()` throws on failure.

== API Endpoints Used

* `GET /nifi-api/flow/processor-types` — List available processor types
* `GET /nifi-api/flow/process-groups/root` — Get root process group
* `GET /nifi-api/process-groups/{id}/processors` — List processors in a group
* `GET /nifi-api/process-groups/{id}/process-groups` — List child process groups
* `POST /nifi-api/process-groups/{id}/processors` — Add processor to canvas
* `DELETE /nifi-api/processors/{id}` — Remove processor from canvas
* `PUT /nifi-api/processors/{id}/run-status` — Start/stop processor
* `GET /nifi-api/processors/{id}` — Get processor details
* `PUT /nifi-api/flow/process-groups/{id}` — Start/stop all processors in a group
* `GET /nifi-api/process-groups/{id}/connections` — List connections in a group
* `POST /nifi-api/flowfile-queues/{id}/drop-requests` — Drop queued FlowFiles
* `DELETE /nifi-api/connections/{id}` — Delete a connection

== Troubleshooting

=== Processor Not Deployed
If `verifyMultiIssuerJWTTokenAuthenticatorIsDeployed()` returns false:

1. Ensure the processor NAR file is installed in NiFi
2. Restart NiFi after installing the NAR
3. Check NiFi logs for deployment errors

=== Authentication Issues (401/403)
1. Ensure `AuthService.ensureReady()` completes before constructing `ProcessorApiManager`
2. API calls use the page's browser context cookies — verify the session is still valid
3. Verify user has permissions to manage processors

=== Connection Deletion Fails (409 Conflict)
The `removeConnectionsForProcessor()` method may return 409 if FlowFiles are queued. It automatically drops queued FlowFiles before retrying the delete. If the error persists, stop the processor first via `stopAllProcessors()`.
