:toc: left

== Overview

WCAG 2.1 Level AA compliance testing for the NiFi JWT Authenticator UI using `@axe-core/playwright` and custom checks.

=== Prerequisites

**CRITICAL**: Accessibility tests require a running NiFi instance with the JWT processor on the canvas.

1. NiFi running at https://localhost:9095/nifi
2. MultiIssuerJWTTokenAuthenticator on the canvas
3. Valid authentication credentials

=== Running Tests

[source,bash]
----
# Quick WCAG compliance check
npm run test:a11y:quick

# Full accessibility test suite
npm run test:a11y

# Generate HTML report
npm run test:a11y:full
----

Tests use a fail-fast approach — they terminate with clear error messages if NiFi or the processor is unavailable.

== Tested Components

The accessibility test suite validates the full custom UI page via axe-core and additionally runs targeted checks on specific components:

[cols="1,2,3"]
|===
|Component |Selector |Test Coverage

|Full Page
|_(entire custom UI frame)_
|axe-core WCAG 2.1 AA scan, custom checks (form labels, keyboard access, focus indicators, ARIA attributes)

|Tab Navigation
|`.jwt-tabs-header`
|Existence verification

|Configuration Form
|`#issuer-config`
|Component-scoped axe-core scan + all custom checks (known pre-existing violation marked as expected failure)
|===

== Writing Accessibility Tests

The existing WCAG compliance spec imports the base `test` fixture (not `accessibilityTest`):

[source,javascript]
----
import { test, expect, takeStartScreenshot } from '../fixtures/test-fixtures.js';

test('Full page WCAG compliance check', async ({ page }) => {
  const accessibilityHelper = new AccessibilityHelper(page);
  const results = await accessibilityHelper.runWCAGCheck();
  expect(results.passed).toBe(true);
});
----

For component-specific checks:

[source,javascript]
----
test('Form accessibility', async ({ page }) => {
  const accessibilityHelper = new AccessibilityHelper(page);
  const result = await accessibilityHelper.checkComponent(
    '#issuer-config',
    'Configuration Form'
  );
  expect(result.passed).toBe(true);
});
----

NOTE: The `accessibilityTest` export from `test-fixtures.js` adds an automatic post-test axe-core scan. The current WCAG spec file uses the base `test` fixture with explicit `AccessibilityHelper` calls for more granular control.

== WCAG 2.1 Level AA Requirements

The framework checks these categories via axe-core (`wcag2aa`, `wcag21aa`, `best-practice` tags):

* **Perceivable** — Color contrast (4.5:1 minimum), alt text, color-independent information
* **Operable** — Keyboard navigation, visible focus indicators, no seizure-inducing content
* **Understandable** — Form labels, error identification, consistent navigation
* **Robust** — Valid markup, proper ARIA usage, screen reader compatibility

== Disabled Rules

Rules that cannot be satisfied in the NiFi custom UI context are disabled globally in `accessibility-helper.js`:

[source,javascript]
----
const DISABLED_RULES = ["bypass", "landmark-one-main", "region"];
----

These rules are disabled because the custom UI runs inside NiFi's iframe and cannot control page-level landmarks.

== Maven Integration

Accessibility tests run via `exec-maven-plugin` in the `integration-test` phase when the `integration-tests` profile is active. Both accessibility and functional tests depend on `self-tests` in the Playwright config. With `workers: 1`, they execute sequentially (not in parallel).

[source,bash]
----
# Full suite via Maven (starts containers, runs all phases, stops containers)
./mvnw clean verify -pl e-2-e-playwright -Pintegration-tests -am
----

== Troubleshooting

=== NiFi Service Not Available
----
CRITICAL FAILURE: NiFi service is not available at https://localhost:9095/nifi
----
Start NiFi: `./integration-testing/src/main/docker/run-and-deploy.sh`

=== JWT Processor Not Found
----
CRITICAL FAILURE: MultiIssuerJWTTokenAuthenticator processor not found on canvas!
----
The processor must be on the canvas. Tests using `processorManager` fixture handle this automatically.

=== Common Fixes

|===
| Issue | Fix

| Color contrast failures
| Ensure 4.5:1 ratio; run `npm run test:a11y -- --grep "color contrast"`

| Missing focus indicators
| Add `outline: 2px solid #0066cc; outline-offset: 2px` on `:focus`

| Form label issues
| Use `<label for="id">` or `aria-label` on every input
|===
