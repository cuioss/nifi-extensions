== Overview

Roundtrip testing workflow: modify code → deploy → test → analyze → repeat.

== Quick Start

[source,bash]
----
# Deploy changes
./integration-testing/src/main/docker/redeploy-nifi.sh

# Run test
cd e-2-e-playwright && npm run playwright:test -- tests/[test-file].spec.js

# Check results
ls -la target/test-results/*/
----

== Workflow Steps

=== 1. Make Code Changes

* UI components: `nifi-cuioss-ui/src/main/webapp/js/`
* Styles: `nifi-cuioss-ui/src/main/webapp/css/`
* HTML: `nifi-cuioss-ui/src/main/webapp/index.html`
* Backend (JWT processor): `nifi-cuioss-processors/src/main/java/`
* Backend (REST gateway): `nifi-cuioss-rest-processors/src/main/java/`
* Backend (UI servlets): `nifi-cuioss-ui/src/main/java/`

=== 2. Build and Deploy

Use the deployment scripts in `integration-testing/src/main/docker/`:

[source,bash]
----
# Initial start (first time or after stop)
./integration-testing/src/main/docker/run-and-deploy.sh

# Quick redeploy (preserves running container)
./integration-testing/src/main/docker/redeploy-nifi.sh

# Full restart (if container issues)
./integration-testing/src/main/docker/stop-test-container.sh
./integration-testing/src/main/docker/run-and-deploy.sh
----

See `CLAUDE.md` "Docker E2E Deployment" section or the `/deploy` skill for the full deployment runbook.

=== 3. Run Tests

==== Maven Orchestration (Recommended)

[source,bash]
----
# Run full integration test suite (self-tests → functional → accessibility)
./mvnw clean verify -pl e-2-e-playwright -Pintegration-tests -am
----

Maven starts containers, runs all Playwright tests sequentially (auth-setup → self-tests → functional → accessibility), and stops containers afterward. Tests run with `workers: 1` since all tests share a single NiFi instance.

==== Direct Test Execution (for development)

[source,bash]
----
# Single test
npm run playwright:test -- tests/specific-test.spec.js

# All tests
npm run playwright:test

# With UI (for debugging)
npm run playwright:test -- --ui

# Self-tests only
npm run test:self
----

=== 4. Analyze Failures

Test artifacts are stored under `target/`:

* `target/test-results/` — Per-test execution artifacts (browser.log, test-run.log, trace.zip), with `screenshots/` and `videos/` subdirectories
* `target/test-results.json` — JSON test results
* `target/junit-results.xml` — JUnit XML results

[source,bash]
----
# Find errors in test logs
find target/test-results -name 'browser.log' -exec grep -n 'ERROR' {} +

# View screenshots from failed tests (macOS: open, Linux: xdg-open)
open target/test-results/screenshots/*.png

# Analyze trace
npx playwright show-trace target/test-results/*/trace.zip
----

=== 5. Fix and Repeat

Based on failure analysis:

* JavaScript errors → Fix in source files
* Missing elements → Update selectors or wait conditions
* 404 errors → Fix resource paths
* Timing issues → Add appropriate waits

Repeat steps 1-4 until tests pass.

== Common Issues

|===
| Issue | Solution

| WebJar 404s
| Fix paths in index.html (remove extra prefixes)

| Element not found
| Verify selector matches rendered HTML

| Timeout errors
| Add waits or increase timeout values

| Port conflicts
| Stop existing containers: `cd integration-testing/src/main/docker && docker compose down -v`
|===

== Debug Techniques

**Interactive debugging**:
[source,javascript]
----
await page.pause(); // Opens Playwright Inspector
----

**Capture intermediate state**:
[source,javascript]
----
await page.screenshot({ path: 'debug.png' });
----

== Postconditions

Before committing, run full validation:

[source,bash]
----
# 1. Pre-commit quality checks
./mvnw -Ppre-commit clean install -DskipTests

# 2. Build entire project with tests
./mvnw clean install

# 3. Run all E2E tests
./mvnw -B --no-transfer-progress clean verify -pl e-2-e-playwright -Pintegration-tests -am

# 4. If all pass → commit
----

All commands must exit with code 0 before committing.
