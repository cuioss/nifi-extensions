How to build, test, and verify the NiFi Extensions project.

== Prerequisites

* Java 21
* Maven (or use the included `./mvnw` wrapper)
* Docker and Docker Compose (for integration and E2E tests)

== Build

[source,bash]
----
./mvnw clean install                    # Full build with tests
./mvnw clean install -DskipTests        # Build without tests
----

== Code Quality

[source,bash]
----
./mvnw -Ppre-commit clean install -DskipTests   # Pre-commit quality checks
./mvnw clean verify -Psonar                      # SonarQube analysis
----

This project follows centralized coding standards from https://github.com/cuioss/cui-llm-rules[cui-llm-rules].

== Integration & E2E Tests

[source,bash]
----
# Start Docker test environment (NiFi + Keycloak)
./integration-testing/src/main/docker/run-and-deploy.sh

# Run Java integration tests via Maven
./mvnw verify -Pintegration-tests -pl integration-testing -am

# Run E2E Playwright tests via Maven
./mvnw verify -Pintegration-tests -pl e-2-e-playwright -am

# Run Playwright E2E tests directly
cd e-2-e-playwright && npm run playwright:test
----

See link:../../integration-testing/README.adoc[Integration Testing] and link:../../e-2-e-playwright/README.adoc[E2E Testing] for detailed instructions.

== PlantUML Diagrams

To regenerate PlantUML PNG images from `.puml` sources (downloads PlantUML JAR automatically if not installed locally):

[source,bash]
----
./doc/plantuml/generate-plantuml.sh
----

This finds all `.puml` files in `doc/plantuml/` and generates corresponding PNG images.
